Quick Start (3 Simple Steps)

Step 1: Open the Newsletter

You can view the newsletter immediately by opening index.html in your web browser.

View Newsletter

Step 2: Customize Your Content

Edit data.json to change the newsletter content:

  • Update headlines and stories
  • Change company information
  • Add your own images
  • Modify letter content

Step 3: Print or Export to PDF

When ready to distribute:

  • Press Ctrl+P (Windows) or Cmd+P (Mac)
  • Select "Save as PDF" as the destination
  • Enable "Background graphics" in print settings
  • Save your professional newsletter!

Running with a Local Server (Recommended)

⚠ Note: For PDF attachments and database features to work properly, you should run a local web server.

Choose one of these methods:

Option 1: Python (Easiest)

If you have Python installed, run this command in the project directory:

# Python 3.x python -m http.server 8000 # Then open: http://localhost:8000

Option 2: Node.js

If you have Node.js installed:

# Install http-server globally npm install -g http-server # Run server http-server -p 8000 # Then open: http://localhost:8000

Option 3: PHP

If you have PHP installed:

php -S localhost:8000 # Then open: http://localhost:8000

Customization Tips

🎨 Change Colors

Edit the CSS variables at the top of styles.css:

--primary-color: #1a472a; /* Main color */ --accent-color: #c9a961; /* Accent color */

🖼️ Add Images

Replace placeholder images in data.json:

  • Cover image: 800×400 pixels (2:1 ratio)
  • Highlight cards: 300×150 pixels
  • Formats: JPG or PNG
  • Keep file sizes under 500KB

📄 Add PDF Attachments

Place your PDF files in the project directory and update data.json:

"pdfAttachments": [ { "title": "Tax Tables", "url": "your-document.pdf", "description": "2026 Tax Reference" } ]

Database Integration

To connect to your database instead of using a static JSON file:

  1. Review database-integration-examples.js for your database type
  2. Create a backend API that returns JSON in the same format as data.json
  3. Update the data source URL in script.js
  4. Test the connection with your server running

Supported Databases

  • MySQL / MariaDB
  • PostgreSQL
  • MongoDB
  • Firebase Firestore
  • Any database with a REST API

See database-integration-examples.js for complete code examples.

Need Help?

Common Issues

  • Images not showing: Check file paths are correct relative to index.html
  • PDF not rendering: Make sure you're running a local server
  • Print issues: Enable "Background graphics" in print settings
  • Data not loading: Check browser console (F12) for errors
